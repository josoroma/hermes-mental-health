# Client-Safe Data Modules in Next.js

## Problem

Server-side modules that import `fs` or `path` cannot be used in
`"use client"` components. Calling `getMeasure(slug)` from `lib/data/measures.ts`
(which does `import fs from "fs"`) in a client component causes:

```
Module not found: Can't resolve 'fs'
```

This happens because `measures.ts` reads JSON templates from disk at runtime,
which only works server-side.

## Solution: Static Import Wrapper

Create a parallel module that imports static JSON at build time:

```typescript
// lib/data/measure-meta.ts — client-safe
import indexData from "@/data/shared/templates/index.json";

const slugToTitle: Record<string, string> = {};
for (const entry of indexData) {
  slugToTitle[entry.slug] = entry.title;
}

export function getMeasureTitle(slug: string): string | undefined {
  return slugToTitle[slug];
}
```

Next.js bundles the JSON at build time via its JSON module support.
No `fs`, no `path` — works in any component.

## When to apply

Any time a client component needs data that originates from a server-only source:

1. Check if the data exists as a static JSON file in the repo
2. Create a lightweight wrapper that imports it at build time via `import data from "@/path/to/file.json"`
3. Build lookup maps (slug→title, id→record) at module init time
4. Export simple getter functions — no `fs`, no `async`, no `path`

## Files using this pattern

- `lib/data/measure-meta.ts` — measure slug→title lookup, `getMeasureMeta()`
- `data/shared/templates/index.json` — the static catalog (generated by `npm run corpus`)
